# Contributing

感谢你愿意改进 find-dupl-file。

## 开发环境

```bash
cargo fmt
cargo test
cargo check
```

提交 PR 前请确保以上命令通过。

## 代码原则

- 保持核心扫描逻辑和 GUI 状态分离。
- 删除文件相关改动必须保守，默认不自动选择保留项。
- 新增核心行为时请补测试。
- 避免提交本地数据库、报告、构建产物或系统临时文件。

## Issue 建议

报告问题时请尽量包含：

- 操作系统
- 运行方式（GUI 或 CLI）
- 比较模式（大小或哈希）
- 相关错误信息
- 可复现步骤

## Pull Request 建议

PR 描述中建议包含：

- 变更目的
- 用户可见影响
- 测试命令和结果
- 是否涉及删除、安全或数据迁移行为
